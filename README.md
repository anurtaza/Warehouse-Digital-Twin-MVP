## Быстрый старт

### Вариант 1 — Docker (рекомендуется)

```bash
docker compose up --build
```

Открыть: http://localhost:3000

### Вариант 2 — Вручную

**Бэкенд:**
```bash
cd backend
pip install -r requirements.txt
python app.py
# Сервер: http://localhost:5000
```

**Фронтенд (в другом терминале):**
```bash
cd frontend
npm install
npm run dev
# UI: http://localhost:3000
```

## Функциональность

### 3D-сцена склада
- Интерактивная визуализация всей карты склада 10×6.
- Ячейки хранилища с динамическим состоянием: заполнена, свободна, горячая зона.
- AGV с визуальным маршрутом и целевой точкой.
- Открытый план склада без лишних перегородок — только стеллажи и коридоры.
- Подсказки по ячейкам и роботам при наведении.

### Маршрутизация AGV
- AGV выбирают цель по всей территории склада.
- Централизованный планировщик строит time-aware A* маршруты и резервирует ячейки во времени.
- Учитываются vertex/edge-конфликты, чтобы два AGV не занимали одну ячейку и не ехали навстречу по одному ребру.
- Визуализация траектории AGV в виде пунктирной линии.
- Плавное движение и корректный поворот AGV по курсу.
- GUI-редактор маршрутов: выберите AGV, включите `Route`, кликните по ячейке склада.
- Операторские команды: `Pause`, `Resume`, `Charge`, `Cancel`.

### Auth / RBAC
- JWT-аутентификация для операторских действий.
- Роли:
  - `manager` — сценарии, оптимизация SKU, управление AGV и назначение маршрутов.
  - `operator` — управление AGV и назначение маршрутов.
  - `logistics` — WMS, заказы, SKU, throughput и отчёты без управления AGV.
  - `viewer` — только просмотр дашборда.
- Demo users:
  - `manager` / `managerpass`
  - `operator` / `operatorpass`
  - `logistics` / `logisticspass`
  - `viewer` / `viewerpass`

### Панели и аналитика
- Режимы пользователя: менеджер склада, оператор, логист.
- Панель метрик в реальном времени: заполненность, активные AGV, заказы, скорость обработки.
- Сценарии: `normal`, `surge`, `agv_fail`, `low_staff`.
- Анализ сценариев и прогнозы производительности.
- WMS-панель с топ-SKU, инвентарем и очередью заказов.
- Экспорт отчёта в JSON.

### Симуляция склада и WMS
- Движок имитирует заполнение и опорожнение ячеек.
- Генерация заказов, смена статусов, синхронизация инвентаря.
- AGV-логика с выбором дальних целей и стабильной сменой маршрута.
- Ручной режим оптимизации размещения SKU по приоритету и расстоянию.
- Система событий с информационными и предупреждающими сообщениями.

## API

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/state` | Полное состояние склада |
| GET | `/api/metrics` | Только метрики |
| GET | `/api/events` | Лог событий |
| POST | `/api/login` | Получить JWT-токен |
| GET | `/api/me` | Проверить текущего пользователя |
| POST | `/api/scenario/{name}` | Переключить сценарий, manager only |
| POST | `/api/optimize` | Оптимизировать размещение SKU, manager only |
| POST | `/api/agv/{id}/command` | Pause/resume/charge/cancel, operator/manager |
| POST | `/api/planner/assign` | Назначить AGV маршрут к ячейке, operator/manager |

**Сценарии:** `normal`, `surge`, `agv_fail`, `low_staff`

**WebSocket события:**
- `state` — полное обновление состояния
- `event` — новое событие в логе

```python
class WarehouseSimulator:
    def get_state(self):
        # Вместо симуляции — запрос к WMS API:
        return wms_client.fetch_current_state()

    def tick(self):
        # Polling реальных данных
        pass
```

Поддерживаемые протоколы подключения:
- REST API WMS (SAP EWM, 1C WMS, Manhattan)
- MQTT/OPC-UA от сенсоров и RFID
- PostgreSQL / ClickHouse для исторических данных
